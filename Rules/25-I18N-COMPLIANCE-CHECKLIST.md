# I18n Compliance Checklist

## 1. 目的

本文件提供 OneLink 在多语言和合规上线前的最终检查清单。

## 2. 文案资源化检查

- [ ] App 用户可见文案已资源化
- [ ] Web 用户可见文案已资源化
- [ ] BFF 未直接返回硬编码中文用户文案
- [ ] 安全、举报、申诉、隐私、删除相关文案均使用 message key
- [ ] `zh-CN` 与 `en` 两套资源齐备

## 3. 语言与地区字段检查

- [ ] `locale` 已独立建模
- [ ] `region` 已独立建模
- [ ] `timezone` 已独立建模
- [ ] `content_language` 已独立建模
- [ ] `notification_language` 已独立建模
- [ ] App/Web 设置页可读写上述字段
- [ ] BFF 能正确透传和落库

## 4. 高风险文案审核检查

- [ ] 拒绝文案已审核
- [ ] 劝导文案已审核
- [ ] 举报文案已审核
- [ ] 申诉文案已审核
- [ ] 隐私文案已审核
- [ ] 数据导出/删除/纠正文案已审核

## 5. 用户数据权利检查

- [ ] 用户可查看关键画像与记忆事实
- [ ] 用户可发起导出请求
- [ ] 用户可发起纠正请求
- [ ] 用户可发起删除请求
- [ ] 这些入口在 App/Web 中可见
- [ ] 后端有真实动作或明确处理流

## 6. 区域与驻留检查

- [ ] 已说明用户区域判定
- [ ] 已说明数据驻留区域
- [ ] 已说明模型调用区域
- [ ] 已说明日志存储区域
- [ ] 已说明跨境依据
- [ ] 已说明区域故障降级方式

## 7. 工程一致性检查

- [ ] OpenAPI / README / 规则文档一致
- [ ] App/Web/BFF 对 locale 字段解释一致
- [ ] i18n key 体系统一
- [ ] 不存在 App 和 Web 各自发明不同 message key 的情况

## 8. 上线阻塞项

若存在以下任一项，不得宣称 Phase 5 完成：

- [ ] 关键用户文案仍硬编码
- [ ] 用户数据权利无真实入口
- [ ] 高风险文案未审核
- [ ] 语言/地区字段仍混用
- [ ] 区域和驻留只有抽象规则没有工程说明

## 9. 审核结论模板

可使用：

- `approved`
- `approved_with_followups`
- `blocked`

并附：

- 审核日期
- owner
- 未完成项
- 修复截止时间
