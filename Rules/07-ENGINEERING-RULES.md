# OneLink Engineering Rules

## 1. 适用范围
本规则适用于 OneLink 全部代码、文档、模型、数据管线、配置和上线流程。

## 2. 核心原则
- 安全优先于速度
- 可审计优先于隐式聪明
- 可回滚优先于一次性大改
- 接口稳定优先于内部完美
- 数据最小化优先于信息贪婪

## 3. 文档规则
- `00-EXECUTIVE-BLUEPRINT.md` 是总蓝图
- 任何重要设计变更必须先更新规范文档，再改代码
- 历史草案不能当作最终需求依据
- 每个新增服务都必须有：
  - 职责说明
  - 输入输出契约
  - 依赖边界
  - 失败处理

## 4. 语言与架构规则

### 4.1 前端
- Web 统一使用 `React + TypeScript`
- 不允许在没有架构评审的情况下引入第二套前端主框架
- 所有跨页面共享状态必须明确来源和生命周期

### 4.2 后端
- `Rust` 为 MVP 默认后端主语言，承担所有核心在线服务
- `Go` 保留给非 MVP 辅助服务和平台工具
- `Python` 不进入高 QPS 在线主链路，只用于 AI 平台和离线智能
- 不允许因个人偏好引入新的主语言

### 4.3 服务边界
- 每个服务只拥有一个明确主职责
- 服务间通信必须走明确 API 或事件契约
- 禁止跨服务直接读对方私有表
- 禁止为省事绕过网关、事件总线或模型网关

## 5. 数据规则

### 5.1 数据最小化
- 默认不采集非必要个人信息
- 所有高敏字段必须说明用途、存储方式和访问控制
- 不允许为了“以后可能有用”提前采集敏感数据

### 5.2 数据分层
- 原始数据、衍生特征、训练样本、公开信息必须分层存储
- 不允许把原始聊天明文直接混入公开展示层
- 不允许把训练数据和生产密钥、生产配置混放

### 5.3 区域和合规
- 所有涉及区域合规的数据都必须带区域属性
- 跨区域复制必须经过审批和审计
- 用户删除、导出、访问审计必须是平台能力

## 6. 模型与 AI 规则

### 6.1 模型访问
- 所有模型调用必须走 `Model Gateway`
- 业务代码不得直接绑定第三方模型 SDK
- 所有 Prompt 必须版本化

### 6.2 AI 生成代码
- AI 可以生成代码、测试、文档和脚本
- 但任何 AI 生成内容都必须经过：
  - 静态检查
  - 测试
  - 安全审查
  - 人类或门禁系统确认
- 禁止直接把 AI 生成代码视为可信生产代码

### 6.3 AI 生成决策
- AI 可以提供推荐和处罚建议
- 但高风险处罚、模型切换、跨区域数据策略不能完全自动执行

## 7. 编码标准

### 7.1 通用
- 代码必须可读
- 命名必须清晰
- 复杂逻辑必须拆分
- 非显然逻辑必须有简短注释
- 禁止保留无用代码、注释掉的大段旧实现和临时调试逻辑

### 7.2 错误处理
- 所有错误都必须显式处理
- 不允许静默吞掉错误
- 面向用户的错误输出不能泄露内部实现和敏感信息

### 7.3 配置管理
- 所有环境差异必须通过配置或部署策略管理
- 不允许把环境路径、密钥、临时开关写死在代码里

## 8. 安全规则

### 8.1 基本要求
- 最小权限
- 默认拒绝
- 明确审计
- 敏感操作双重确认

### 8.2 账户与权限
- 管理操作必须有权限校验
- 高危操作必须有审计日志
- 生产权限和开发权限分离

### 8.3 内容安全
- 找人请求、私信、主页和举报都必须进入治理链路
- 禁止通过任何技术旁路绕过审核和投诉系统

## 9. 测试规则

### 9.1 必须具备
- 单元测试
- 契约测试
- 集成测试
- 关键主流程 E2E 测试

### 9.2 重点覆盖
- 注册登录
- 聊天主链路
- 找人请求
- 风险拦截
- 推荐结果
- 陌生人首条私信限制
- 投诉和处罚链路

### 9.3 模型与策略测试
- Prompt 变更必须有回归测试
- 排序策略变更必须有离线对比
- 风险模型变更必须验证误伤率和漏判率

## 10. 可观测性规则
- 所有服务必须输出结构化日志
- 所有主链路必须有追踪
- 所有关键指标必须入监控
- 新服务没有健康检查和核心指标，不允许上线

## 11. 发布规则

### 11.1 合入门槛
- 代码审查通过
- 自动化测试通过
- 安全扫描通过
- 文档已更新

### 11.2 上线门槛
- 有回滚方案
- 有监控指标
- 有值班和告警接收
- 涉及模型或策略的改动必须先灰度

### 11.3 禁止事项
- 禁止周五晚大规模无灰度上线
- 禁止未评估成本直接扩大模型流量
- 禁止未审计的脚本直连生产核心资源

## 12. 数据与模型实验规则
- 所有实验必须有唯一 ID
- 所有实验必须记录：
  - 假设
  - 数据集版本
  - 模型版本
  - 指标结果
  - 是否上线
- 没有记录的实验视为无效，不可用于发布决策

## 13. AI Agent 工作流规则
- Agent 可以分工，但职责必须清晰
- Agent 不得自己新增高权限能力
- Agent 预算必须受控
- 关键变更必须经过审批
- Agent 产生的任务、代码、测试、审计结果都必须可追溯

## 14. 禁止清单
- 禁止抓取未经同意的个人外部隐私数据
- 禁止把灰区找人需求包装成正常功能
- 禁止把内部风险标签对外展示
- 禁止绕过用户可见的边界设置
- 禁止把投诉系统当摆设
- 禁止以“AI 判断”作为所有争议的最终结论

## 15. 执行方式
- 所有仓库、服务、模型和后台功能都必须遵守本规则
- 规则若需变更，必须先更新本文件，再更新相关实现
