# OneLink Constitution

## 1. 根定义

OneLink 是一个以 AI 好朋友为入口、以长期理解人为核心、以安全连接真实人为目标的全球社交平台。

它不是普通聊天机器人，也不是普通推荐流。OneLink 的长期竞争力来自：更会理解用户、记住关键、忘掉噪音，并在合规边界内帮助用户连接合适的人。

## 2. 当前工程状态判断

截至 2026-05-15，项目处于：

- 规划资料完整度高。
- Rust 工程骨架已铺开。
- `chat -> memory -> profile` 第一条纵切面已有可联调雏形。
- 账号安全、持久化、测试、CI/CD、前端产品化和部署仍明显不足。

下一阶段重点不是继续扩大战线，而是把第一条纵切面做成可验证、可部署、可演进的核心系统。

## 3. 不可变铁律

### 3.1 单一主写权

每类主数据只能有一个 owner service。跨服务读可以通过内部接口或只读投影完成，跨服务写必须走 owner 服务。

### 3.2 Memory 不越权

`context-service` 是 Memory Compute Layer，不是业务真相库。它可以抽取、压缩、检索、整合记忆，并发起画像投影请求，但不能直接写画像主表、推荐结果或处罚结论。

### 3.3 同步负责响应，异步负责演化

在线同步链路只做会话校验、最小上下文组装、模型调用和响应返回。记忆抽取、画像投影、向量更新、策略优化和复杂评估必须优先异步化。

### 3.4 逻辑 Agent 常驻，运行时 Agent 按需唤醒

每个用户可以拥有终身 `agent_id` 和无限可回看时间线，但不能实现成每用户常驻进程，也不能每次推理加载全部历史。

### 3.5 选择性遗忘是核心能力

系统不追求记住一切。长粘贴、低价值闲聊、通用知识问答、过期偏好必须经过价值评分、压缩、冷热分层和可追溯遗忘。

### 3.6 AutoResearch 优化 Policy，不修改 Constitution

AutoResearch 可以优化阈值、权重、排序、召回、上下文预算、遗忘策略和追问节奏，但不能自动修改服务边界、DDL、OpenAPI、事件 schema、主写权、合规底线或 Lumi 人格宪法。

### 3.7 高风险优化必须可回放、可灰度、可回滚

任何进入主链路的自动策略优化，必须经过 offline replay、shadow、canary、rollback 四道门禁。

### 3.8 Rust 是在线核心主语言

面向全球几十亿用户的高并发目标，在线核心服务优先使用 Rust。Go 用于配套平台、边缘工具、运维控制面、异步 worker 或生态适配，不抢占 Rust 核心服务主线。

### 3.9 规则必须级联到工程资产

高层规则更新后，必须检查并同步：`repo/platform/contracts/`、`repo/data-platform/`、`repo/services/*/README.md`、`repo/tests/`、脚本和 CI。禁止只改规则不改契约。
